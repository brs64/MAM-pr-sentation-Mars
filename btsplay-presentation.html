Projet SAE - BUT Informatique 3 - IUT de Bayonne

Le Contexte

Le BTS Audiovisuel de Biarritz produit quotidiennement des contenus vidéo professionnels stockés sur deux serveurs NAS distincts

NAS MXF

Format broadcast professionnel - Qualité maximale pour le montage

NAS MP4

H.264 25Mbps - Diffusion et consultation web

Mission : Créer une plateforme unifiée permettant aux étudiants et enseignants d'accéder, rechercher et gérer facilement ces contenus vidéo

⬇️ Naviguer vers le bas pour les détails de chaque service

btsplay-laravel

Rôle : Cœur de l'application web
Technologies : Laravel 12, PHP 8.2+, Eloquent ORM
Responsabilités :

  • Gestion des routes et contrôleurs
  • Authentification et autorisation (Laravel Breeze + Spatie)
  • Interaction avec la base de données MySQL
  • Communication avec les serveurs FTP
  • Génération de miniatures et streaming HLS

btsplay-queue

Rôle : Worker pour jobs asynchrones
Technologies : Laravel Queue (database driver)
Responsabilités :

  • Traitement des jobs de transcodage FFAStrans
  • Surveillance de l'état des conversions vidéo
  • Gestion des transferts entre NAS
  • Exécution de tâches longues en arrière-plan
  • Réconciliation automatique des contenus

btsplay-nginx

Rôle : Serveur web reverse proxy
Technologies : Nginx (Alpine)
Responsabilités :

  • Exposition de l'application sur le port 8080
  • Proxy des requêtes vers PHP-FPM
  • Gestion des fichiers statiques (CSS, JS, images)
  • Optimisation des performances avec cache
  • Configuration SSL/TLS pour production

btsplay-mysql

Rôle : Base de données relationnelle
Technologies : MySQL 5.7
Contenu :

  • Table medias : métadonnées des vidéos
  • Tables users, eleves, professeurs
  • Tables roles et permissions (Spatie)
  • Table jobs : queue de transcodage
  • Migrations Laravel pour versioning du schéma

btsplay-ftp-pad

Rôle : Serveur FTP pour fichiers MXF
Technologies : Pure-FTPd
Caractéristiques :

  • Port : 21001
  • Stockage : fichiers MXF broadcast (qualité maximale)
  • Utilisé par les équipes de production
  • Format professionnel pour montage vidéo
  • Accès sécurisé avec authentification

btsplay-ftp-arch

Rôle : Serveur FTP pour fichiers MP4
Technologies : Pure-FTPd
Caractéristiques :

  • Port : 21002
  • Stockage : fichiers MP4 H.264 25Mbps
  • Format optimisé pour consultation web
  • Utilisé pour streaming HLS
  • Accessible aux étudiants et professeurs

Workflow de Transcodage

Processus automatique :

  1. L'utilisateur demande une vidéo manquante sur un NAS
  2. Laravel crée un job dans la queue
  3. Le worker btsplay-queue récupère le job
  4. Appel API vers FFAStrans pour transcodage
  5. Surveillance de l'état du job (polling)
  6. Récupération du fichier converti
  7. Mise à jour de la base de données
  8. Notification à l'utilisateur

Cas d'Usage Concrets

Étudiant en production

Un étudiant termine son tournage. Il dépose le fichier MXF sur le NAS. BTSPlay l'indexe automatiquement, génère une miniature, et le transcrit en MP4 pour consultation web. L'étudiant peut ensuite partager le lien à ses professeurs.

Professeur en évaluation

Un professeur recherche tous les projets d'une promotion. Il utilise les filtres de recherche, visionne directement les vidéos dans le navigateur, et laisse des commentaires sur les métadonnées.

Administrateur en maintenance

L'admin constate qu'un NAS manque de fichiers. Il lance une réconciliation qui identifie les vidéos manquantes et déclenche automatiquement les transcodages nécessaires.

⬇️ Scénarios avancés

Scénarios Avancés

Import massif de promotion :

L'admin importe un CSV avec 50 étudiants. Le système crée automatiquement les comptes, attribue les rôles, et envoie les emails de bienvenue.

Archivage de fin d'année :

Tous les projets d'une promotion sont marqués comme archivés. Ils restent accessibles en consultation mais ne sont plus modifiables.

Maintenance planifiée :

Une synchronisation automatique nocturne vérifie l'intégrité des fichiers sur les deux NAS et alerte en cas de divergence.

Défis Techniques Relevés

Synchronisation bi-directionnelle

Réconcilier deux NAS avec des fichiers dans différents formats et éviter les doublons

Streaming vidéo performant

Segmentation HLS à la volée pour des fichiers de plusieurs Go sans saturer le serveur

Gestion des jobs asynchrones

Transcodage de vidéos longues sans bloquer l'interface, avec gestion des échecs et reprises

Permissions granulaires

Système de rôles complexe permettant un contrôle fin des accès selon le profil utilisateur

Nos Solutions

Pour la synchronisation :

  • Notifications en temps réel de l'avancement
